# wordle

Инструмент для подбора слов в русской версии Wordle.

Проект умеет:

- автоматически симулировать партии и оценивать качество стратегии;
- подсказывать следующие слова во время реальной внешней игры;
- фильтровать словарь по ответам игры;
- выбирать следующее слово случайно или с учетом вероятностей букв по позициям.

## Как устроен проект

Основные файлы:

- `main.py`:
  содержит интерфейс запуска, локальную модель игры и два режима работы;
- `predictor.py`:
  содержит класс `FindAnswer`, который хранит текущее состояние, фильтрует словарь и предлагает следующее слово;
- `rus_words_no_duplicates.txt`:
  словарь русских слов, который используется как база кандидатов.

## Формат ответа игры

Состояния букв:

- `-` — буквы нет в слове;
- `*` — буква есть в слове, но стоит не на своем месте;
- `+` — буква стоит на правильном месте.

Пример:

- слово: `лампа`
- ответ: `-*+-*`

Это означает, что:

- первая буква отсутствует в ответе;
- вторая буква есть, но стоит не там;
- третья буква угадана по позиции;
- и так далее.

## Режимы работы

### 1. Симуляция игры

Режим `-model-game` запускает серию автоматических игр.

Что происходит:

1. Из словаря выбирается случайное секретное слово.
2. Алгоритм начинает угадывать его.
3. После каждой попытки список допустимых слов сокращается.
4. По итогам серии печатается статистика.

Пример запуска:

```bash
python3 main.py -model-game
```

Запуск 100 игр:

```bash
python3 main.py -model-game -N 100
```

Выводит:

- процент побед;
- среднее число попыток;
- при желании список неугаданных слов и последних попыток.

Дополнительные флаги:

- `-N <число>` — количество симуляций;
- `-random` — использовать полностью случайный выбор следующего слова из текущего набора кандидатов;
- `-logs` — вывести слова, которые алгоритм не смог угадать, и последнюю догадку для каждого случая.

### 2. Подсказки для внешней игры

Режим `-outer-game` предназначен для игры вне программы, когда вы сами вводите ответ Wordle после каждой попытки.

Пример запуска:

```bash
python3 main.py -outer-game
```

Пример из исходного проекта:

```bash
python3 main.py -outer-game -regime e
```

Во время работы программа:

1. показывает количество доступных слов;
2. предлагает следующее слово;
3. просит подтвердить его;
4. принимает ответ игры и пересчитывает возможные варианты.

Поддерживаемые флаги:

- `-regime auto` — в качестве слова для обработки используется предложенное программой слово;
- `-regime <любое_другое_значение>` — программа дополнительно спросит слово вручную;
- `-random` — использовать случайную стратегию вместо вероятностной.

Поведение `-regime`:

- `auto` удобно, если вы всегда играете предложенным словом;
- любой другой режим полезен, если вы иногда вводите в игре другое слово и хотите вручную сообщить его программе.

## Стратегии выбора слова

Поддерживаются две стратегии.

### Случайная стратегия

Метод `get_next_word()` выбирает случайное слово из текущего списка допустимых кандидатов.

Особенности:

- простая;
- быстрая;
- не учитывает частоты букв.

### Вероятностная стратегия

Метод `get_clever_random_word()` сначала оценивает вероятности букв по позициям среди оставшихся слов, а затем выбирает слово с учетом этих весов.

Идея стратегии:

- чем чаще буква встречается в конкретной позиции среди оставшихся кандидатов, тем выше вклад этой буквы в оценку слова;
- итоговый вес слова вычисляется как произведение позиционных вероятностей его букв;
- выбор остается случайным, но хорошие кандидаты выпадают чаще.

Эта стратегия используется по умолчанию.

## Как работает фильтрация слов

После каждого ответа игры алгоритм обновляет ограничения:

- для `+` фиксирует букву в конкретной позиции;
- для `*` запрещает букву в текущей позиции, но требует ее наличие в слове;
- для `-` удаляет букву из допустимых вариантов.

Далее:

- строится регулярное выражение по допустимым буквам в каждой позиции;
- из текущего словаря отбираются слова, которые подходят под позиционные ограничения;
- затем отбрасываются слова, в которых отсутствуют уже подтвержденные буквы.

## Сброс состояния

Метод `reset()` у `FindAnswer`:

- очищает историю предыдущих слов;
- очищает историю ответов;
- сбрасывает известные буквы;
- возвращает полный список слов-кандидатов;
- сбрасывает внутренние вероятности.

Этот метод используется между симуляциями в режиме `-model-game`.

## Ограничения текущей реализации

Важно учитывать особенности алгоритма:

- проект рассчитан на слова фиксированной длины 5 букв;
- словарь загружается из локального файла `rus_words_no_duplicates.txt`;
- буква `ё` нормализуется в `е`;
- логика обработки повторяющихся букв упрощена и может отличаться от поведения оригинального Wordle;
- зависимости `numpy` и `tqdm` используются в коде, но должны быть доступны в окружении при запуске.

## Требования

Проект использует Python 3.8+.

В `pyproject.toml` указаны настройки Poetry, но для фактического запуска также нужны библиотеки:

- `numpy`;
- `tqdm`.

Если вы запускаете проект вручную без Poetry, убедитесь, что они установлены.

## Быстрые примеры

Запустить 10 автоматических игр:

```bash
python3 main.py -model-game
```

Запустить 500 автоматических игр со случайной стратегией:

```bash
python3 main.py -model-game -N 500 -random
```

Запустить внешний режим с выводом подсказок:

```bash
python3 main.py -outer-game
```

Запустить внешний режим и вручную вводить фактически сыгранное слово:

```bash
python3 main.py -outer-game -regime manual
```

## Что можно улучшить

Потенциальные направления развития:

- более точная обработка повторяющихся букв;
- отдельный список стартовых слов;
- выбор лучшего слова по информации, а не только по вероятности;
- нормальное описание зависимостей в `pyproject.toml`;
- тесты для проверки фильтрации и корректности ответов.
